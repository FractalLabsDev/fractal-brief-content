# Fractal Brief Content

Markdown content for [brief.fractallabs.dev](https://brief.fractallabs.dev).

## Structure

Each `.md` file in the root directory is accessible via its filename (without extension) as a URL slug.

For example:
- `hello.md` -> `brief.fractallabs.dev/hello`
- `proposal.md` -> `brief.fractallabs.dev/proposal`

## Creating a Brief

1. Create a new `.md` file with your desired slug name
2. Commit and push to `main`
3. Access at `brief.fractallabs.dev/<slug>`

## Format

Standard GitHub-flavored Markdown is supported, including:
- Headers (h1-h3)
- Lists (ordered and unordered)
- Code blocks with syntax highlighting
- Tables
- Links and images
- Blockquotes
